## Zero-shot の例

```
以下のイタリアンレストランの利用者のレビューを「ポジティブ」か「ニュートラル」か「ネガティブ」に分類してください。

利用者のレビュー
このレストランは料金は高かったが、料理美味しかった。
```

## Few-shot の例

* **Zero-shot での判定が「ニュートラル」が多い場合**
```
以下のイタリアンレストランの利用者のレビューについて例を参考にして「ポジティブ」か「ニュートラル」か「ネガティブ」に分類してください。

利用者のレビュー
このレストランは料金は高かったが、料理は美味しかった。

例）
利用者のレビュー
このレストランは騒がしい雰囲気だったが料理は美味しかった。
結果）「騒がしい雰囲気だったが」という文に続いて最後に「美味しかった」とポジティブな意見で締めくくっているので「ポシティブ」と判定できる
```

* **Zero-shot での判定が「ポシティブ」が多い場合**
```
以下のイタリアンレストランの利用者のレビューについて例を参考にして「ポジティブ」か「ニュートラル」か「ネガティブ」に分類してください。

利用者のレビュー
このレストランは料金は高かったが、料理は美味しかった。

例）
利用者のレビュー
このレストランは騒がしい雰囲気だったが料理は美味しかった。
結果）「騒がしい雰囲気だったが」という文に続いて最後に「美味しかった」とポジティブな意見で締めくくっているが、ネガティブ評価とポジティブ評価の両方があるので「ニュートラル」と判定できる
```

## Chain-of-thought (CoT)  の例

### 例 1

(1) CoT を使わない場合
```
あなたは北を向いて立っています。左に90度回転し、後ろを向き（180度回転）、さらに右に270度回転しました。あなたの右手側はどの方角ですか？回答だけを提示して下さい。
```

(2) CoT を使う場合
```
あなたは北を向いて立っています。左に90度回転し、後ろを向き（180度回転）、さらに右に270度回転しました。あなたの右手側はどの方角ですか？段階的に考えて回答を提示して下さい。
```

* 想定する回答
    - (1) は 南西、西のような誤答の可能性が高くなる
    - (2) は 正解（東）の可能性が高くなる

### 例 2
(1) CoT を使わない場合
```
AはBより背が高い。CはBより背が低い。DはCより背が高いがBより背が低い。
この4人を背の高い順に並べてください。答えだけを記号で。
```

(2) CoT を使う場合
```
AはBより背が高い。CはBより背が低い。DはCより背が高いがBより背が低い。
この4人を背の高い順に並べてください。ステップバイステップで考えてから答えてください。
```

* 想定する回答
    - (1) は A > D > B > C のような誤答の可能性が高くなる
    - (2) は A > B > D > C と正解の可能性が高くなる
